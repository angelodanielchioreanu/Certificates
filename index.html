    Project Description
    3Di is a software development company which designs and implements web and mobile applications for City of Los Angeles offices like Fire Department and Department of Water and Power. Such apps make use of editable maps, photos and charts which are processed by running Azure Cloud Services. I have designed and implemented a common logging system based on RESTful API hosted in an Azure cloud service, consumed by all other web servers and client applications. The log records can help the administrator provide customer support, defect maintenance and management estimations about the usage patterns and preferences related to the user experience. The common logger clients send execution records to the cloud service in a similar way with log4net library, providing log levels, timestamps, code messages, serialized exceptions and execution thread context. I implemented the server logger with C# ASP.NET for Web API 2 and CORS and used Entity Framework with stored procedures for MS SQL database. Implemented with Angular and Bootstrap a log viewer, grid-based having sortable columns and automatic data refresh, using AJAX and having the pagination prepared on the server side in stored procedures, to be very fast. Implemented two types of client-loggers (libraries) for consumers; one in JavaScript, object-oriented, for HTML web client-type and one in C#.NET for web server-type or desktop applications. For the server-type implementation of the logger, I created a resource lifetime management with synchronization in multithreading to queue the logging requests from its clients. Used TPL (Task Parallel Library) and HTTPClient from .NET to send asynchronously the log records over HTTP. The project relied on Azure Continuous Integration (code check-ins triggered the building and deployment of the project in the cloud). I created the architecture documentation with block, sequence and class diagrams.
